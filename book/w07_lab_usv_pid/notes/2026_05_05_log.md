Notes for lab procedure:

* Making sense of the vehicle logs can be tricky.  We recommend the following process
    * Each time the pilot arms the autopilot it starts a new log file.  Think of this as one run of your experiment.   So you will need to correlate each log file with what was being tested and the corresponding set of experiment parameters.
    * When the pilot arms, the scribe notes the UTC time (with seconds) and a description of the purpose of the experiment, e.g, "Baseline, run #2, didn't work".   When the log file is post-processed to MATLAB format, the UTC time is appended to the file name.  You will have 10's of log files, so knowing which to focus on 
    * When the pilot switches the autopilot into ACRO mode (engaging the feedback controllers), the planner confirms ACRO mode in Mission Planner before the pilot executes the test.
    * When the experiment is over, pilot de-ARMs and re-ARMs the autopilot to close the experiment.  This keeps the time series you will be processing in MATLAB easier to visualize.

* The MATLAB scirpt description - prints param values for the PID
